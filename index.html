Privacy Policy

Application Name: Man Down
Last update: March 2026

Introduction

This Privacy Policy describes how the Man Down application processes and protects user data when installed on devices compatible with the Garmin Connect IQ platform.

The application provides a Man-Down monitoring system designed to detect falls or emergency situations and to notify a remote monitoring operator.

The system has been designed according to privacy by design and data minimization principles, in line with the General Data Protection Regulation (GDPR).

Data Collected

The application may collect the following data from the smartwatch sensors:

Motion data

  • Accelerometer measurements

Physiological data

  • Heart rate

Location data

  • GPS position

Device-related data

  • Device identifier

Application status data

  • Monitoring status
  • Alarm status
  • System diagnostic information

The application does not collect personal identifiers such as:

  • name
  • email address
  • phone number
  • contacts
  • calendar information

Purpose of Data Processing

Data are processed exclusively for the following purposes:

  • detection of potential critical events (fall/prolongued immobility)
  • monitoring of abnormal physiological conditions
  • emergency alert management
  • transmission of emergency information to a monitoring service
  • system validation and performance optimization, only where explicitly consented by the user

Data are not used for advertising, marketing, or user profiling.

Data Transmission

Some data may be transmitted from the smartwatch, through the Garmin Connect mobile application, to a remote server infrastructure.

The transmitted data may include:

  • device identifier
  • GPS location
  • heart rate
  • monitoring status
  • alarm events

Accelerometer data may be transmitted for algorithm validation purposes only where the user has explicitly provided consent.

All communications are performed using secure network protocols (TLS/HTTPS).

Local Data Storage

The application performs limited local persistent storage only when data transmission to the remote server is not successful.

Locally stored data may include:

  • undelivered event messages

This storage is used to:

  • ensure reliable delivery of critical information
  • maintain system continuity in case of temporary connectivity issues

Data are:

  • limited strictly to undelivered messages
  • automatically deleted once successfully transmitted
  • not stored continuously or redundantly

Remote Data Processing

When the system is connected to the internet via the Garmin Connect mobile application, selected data may be transmitted to a remote server to support:

  • emergency monitoring
  • incident analysis
  • system performance improvement (only where explicitly consented by the user)

Location data may be periodically acquired and transmitted by the application to maintain an up-to-date estimate of the user’s position.

This approach allows the system to provide the most accurate available location in the event of a detected fall or a manually triggered SOS alert, including scenarios where a real-time GPS fix may not be immediately available.

Outside emergency conditions, location data are transmitted but not persistently stored or further processed, and are not used for tracking or profiling purposes.

Persistent storage and processing of location data occur only in relation to confirmed emergency events.

Access to monitoring data is restricted to authorized operators.

Data Security

Appropriate technical and organizational measures are implemented to protect personal data, including:

  • encrypted communication channels (HTTPS/TLS)
  • access control mechanisms
  • data minimization principles
  • controlled access to monitoring infrastructur

User Rights

Users may request:

  • access to their data
  • deletion of stored data
  • withdrawal of consent for optional data processing

Requests can be sent to the contact address listed below.
